# Улучшения экрана расписания: навигация, реактивное время, типы, состояния

## Problem Frame

Экран расписания после первой реализации имеет четыре конкретных слабости:

1. **Нет навигации по неделям** — пользователь видит только текущую неделю и не может планировать вперёд или назад.
2. **Stale active-state** — `LessonCard` вычисляет `isActive`/`isPast` один раз при рендере; прогресс-бар не движется, значок "Сейчас идёт" не гаснет по окончании пары.
3. **`type: String`** — тип пары как голая строка порождает разброс логики по виджетам; добавление нового типа требует обновления нескольких мест.
4. **Одинаковые пустые состояния** — выходной день, свободный день и сетевая ошибка показывают одинаковое "Нет пар", подрывая доверие к данным.

## Requirements

### R1. Навигация по неделям
- R1.1. WeekStrip показывает иконку-стрелку `‹` слева и `›` справа от ряда чипов.
- R1.2. Тап по стрелке сдвигает отображаемую неделю на 7 дней назад/вперёд.
- R1.3. Горизонтальный свайп по WeekStrip выполняет то же действие (свайп влево → следующая неделя, вправо → предыдущая).
- R1.4. При смене недели автоматически выбирается первый день новой недели (понедельник), если ранее выбранный день не попадает в неё.
- R1.5. Стрелка `‹` скрывается (или неактивна) когда дата ушла достаточно далеко в прошлое (за пределы данных); аналогично `›` — в будущее. Точный предел определяется при планировании.
- R1.6. Переход обратно к текущей неделе: тап на любой чип с сегодняшней датой, либо отдельная кнопка "Сегодня" (решить при планировании).

### R2. `nowProvider` — реактивное время
- R2.1. Существует единый реактивный источник текущего времени, доступный всем виджетам через Riverpod.
- R2.2. `LessonCard` переключается в состояние `active`/`past` автоматически при наступлении соответствующего времени — без перезапуска приложения.
- R2.3. `LessonProgressBar` отображает актуальный прогресс, обновляясь с разумной периодичностью (не реже 1 раза в минуту).
- R2.4. `SyncStatusBar` показывает актуальное текущее время (тикает).
- R2.5. Реактивное время не влияет на выбранный день (`selectedDayProvider`) и не перебрасывает пользователя на "сегодня" автоматически.

### R3. `LessonType` enum
- R3.1. Тип пары представлен в domain-слое как перечисление, а не строка.
- R3.2. Поддерживаемые значения: лекция, практика, лабораторная, другое (fallback).
- R3.3. Каждый тип несёт свой отображаемый ярлык (ЛЕК / ПР / ЛАБ / ?) и цвет для бейджа.
- R3.4. Неизвестная строка при разборе маппируется в fallback-тип без исключения.
- R3.5. Все потребители типа (`_TypeBadge` и др.) используют значения из enum, без хардкода строк.

### R4. Три различных пустых состояния
- R4.1. **Выходной день** (суббота / воскресенье): иллюстративное сообщение "Выходной — отдыхай!". Никакой кнопки Retry.
- R4.2. **Свободный день** (будний, пар нет): "Нет пар — свободный день". Никакой кнопки Retry.
- R4.3. **Ошибка загрузки**: сообщение об ошибке (тип ошибки: нет сети / ошибка разбора / нет кэша) и кнопка **"Повторить"**, которая инициирует повторный запрос данных.
- R4.4. Определение выходного дня производится по `selectedDay.weekday` (6 = суббота, 7 = воскресенье).

## Success Criteria

- Пользователь может просматривать расписание следующей и предыдущей недели тапами и свайпом.
- Прогресс-бар активной пары обновляется без перезапуска приложения.
- `LessonCard` меняет визуальное состояние (активная → завершённая) в нужный момент автоматически.
- При сетевой ошибке + пустом кэше показывается кнопка Retry, а не просто "Нет пар".
- Добавление нового типа пары требует изменения только enum и маппинга — не виджетов.
- `flutter analyze` проходит без ошибок.

## Scope Boundaries

- Без анимации перехода между неделями в списке пар (только в WeekStrip).
- Без кэширования нескольких недель в моках — другие недели могут показывать пустые дни.
- Без фильтрации по `LessonType` на этом экране (список/поиск — отдельная задача).
- Без смены темы или локализации (за рамками этой задачи).
- Без auto-scroll к активной паре (deferred, требует `nowProvider` как зависимость — следующий шаг).

## Key Decisions

- **Свайп + стрелки для навигации**: наиболее натуральное мобильное взаимодействие + визуальная affordance.
- **Единый `nowProvider`**: один источник времени для всех виджетов — тестируемость через Riverpod overrides, нет рассинхронизации между `LessonCard` и `SyncStatusBar`.
- **`LessonType` в domain**: тип — доменная концепция, не UI-строка. Виджеты зависят от цвета/лейбла из enum, не хардкодируют их.
- **Три состояния**: выходной vs свободный день — разные сообщения, т.к. информационная ценность разная. Кнопка Retry только при ошибке.

## Dependencies / Assumptions

- `selectedDayProvider` уже существует и работает — навигация по неделям добавляет `currentWeekProvider` поверх него.
- `ScheduleRepositoryImpl` с моками достаточен — реальный API не нужен для этих улучшений.
- `Failure` sealed class уже имеет `NetworkFailure`, `ParseFailure`, `CacheFailure` — можно использовать в R4.3.
