# Profile & Onboarding

## Problem Frame

Приложение не знает, чьё расписание показывать: нет выбора группы/преподавателя. При первом запуске пользователь должен пройти онбординг и задать профиль. После — профиль доступен для редактирования во вкладке «Профиль».

## Requirements

- R1. При первом запуске (SharedPreferences-флаг `onboarding_complete` отсутствует или `false`) приложение направляет пользователя на экран онбординга вместо главного экрана.
- R2. Онбординг предлагает выбор режима: **Студент** или **Преподаватель**.
- R3. В режиме **Студент** пользователь последовательно выбирает: группу → подгруппу (1 или 2) → вводит имя (опционально).
- R4. В режиме **Преподаватель** пользователь выбирает преподавателя из списка.
- R5. Список групп и преподавателей загружается из YSTU API и кэшируется локально в Drift.
- R6. Если при онбординге нет сети и список не загрузился — показывается состояние ошибки с кнопкой «Повторить». Продолжить без выбора нельзя.
- R7. После завершения онбординга выставляется флаг `onboarding_complete = true`, профиль сохраняется, пользователь переходит на вкладку «Расписание».
- R8. Вкладка «Профиль» отображает текущий профиль (режим, группа/преподаватель, подгруппа, имя) и позволяет изменить любое поле, включая смену режима.
- R9. При смене группы или режима расписание перезагружается из API для нового субъекта.

## Success Criteria

- Новый пользователь не может попасть в расписание без выбора группы/преподавателя.
- После онбординга расписание загружается именно для выбранной группы или преподавателя.
- Изменение профиля во вкладке «Профиль» немедленно отражается в расписании.

## Scope Boundaries

- Авторизация и личный кабинет ЮРГТУ — вне scope.
- Несколько профилей / переключение между профилями — вне scope.
- Push-уведомления, связанные с профилем, — вне scope.

## Key Decisions

- **Два режима (студент / преподаватель) вместо единого поля**: позволяет иметь разный набор полей и разные API-запросы для каждого режима.
- **Блокировка онбординга без сети**: без группы приложение бессмысленно; пустой кэш расписания лучше не показывать.
- **Хранение профиля в SharedPreferences**: профиль — плоская структура (≤ 5 полей), Drift избыточен; SharedPreferences достаточно.

## Dependencies / Assumptions

- API ЮРГТУ предоставляет эндпоинты `/groups` и `/teachers` (описаны в `2026-04-17-api-client-requirements.md`).
- `api_client.dart` должен быть реализован до начала работы над онбордингом (или реализовываться параллельно).
- Кэш групп и преподавателей хранится в Drift (таблицы `groups`, `teachers`).
