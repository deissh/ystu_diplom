# API-клиент: расписание ЯГТУ

## Problem Frame

Приложение работает только на захардкоженных мок-данных. Нужно подключить реальный REST API ЯГТУ (`gg-api.ystuty.ru`) для получения списка групп, списка преподавателей и расписания. Пользователь выбирает свою группу в настройках и видит актуальное расписание.

## API endpoints

| Назначение | URL |
|---|---|
| Список групп | `GET /s/schedule/v1/schedule/actual_groups` |
| Список преподавателей | `GET /s/schedule/v1/schedule/actual_teachers` |
| Расписание группы | `GET /s/schedule/v1/schedule/group/{groupName}` |
| Расписание преподавателя | `GET /s/schedule/v1/schedule/teacher/{teacherId}` *(нужна верификация)* |

### Ключевые поля ответа расписания

- `items[]` — массив недель → дней → занятий
- `lesson.lessonName` — название дисциплины
- `lesson.teacherName`, `lesson.teacherId`, `lesson.auditoryName`
- `lesson.startAt` / `lesson.endAt` — UTC ISO-строки
- `lesson.type` — число: 2=поток (лекция), 4=семинар, 8=семинар без потока
- `lesson.isLecture` — булево; дублирует `type==2`
- `lesson.parity` — 0=каждую неделю, 1=нечётная, 2=чётная

## Requirements

- **R1.** `ApiClient` реализует три публичных метода: `fetchGroups()`, `fetchTeachers()`, `fetchGroupSchedule(String groupName)`. Если решение по расписанию преподавателя подтверждено — добавить `fetchTeacherSchedule(int teacherId)`.
- **R2.** `Parser` преобразует JSON-ответы в data-модели: `GroupListModel`, `TeacherModel`, `ScheduleWeekModel` (→ `ScheduleDayModel` → `LessonModel`).
- **R3.** Список групп кешируется локально — он нужен для пикера в экране настроек. Группы структурированы по институту (`name` + `groups: List<String>`).
- **R4.** Список преподавателей кешируется локально — нужен для пикера при выборе преподавателя. Поля: `id`, `name`.
- **R5.** Расписание кешируется в Drift (существующий механизм). При отображении занятия фильтруются по номеру выбранной недели с учётом чётности: показываются занятия с `parity == 0` (всегда) и занятия, чья чётность совпадает с чётностью номера недели.
- **R6.** `LessonType` маппится из API: `isLecture == true` → `lecture`; `type == 4` → `practice`; `type == 8` → `lab`; иначе → `other`.
- **R7.** `ScheduleRepositoryImpl.getSchedule()` и `watchSchedule()` переключаются с мок-данных на реальный API + Drift.
- **R8.** Ошибки сети не блокируют UI — при недоступности API возвращаются кешированные данные + `NetworkFailure` (существующий паттерн).

## Success Criteria

- Приложение показывает реальное расписание группы ЦИС-47 (или любой другой выбранной группы) после первой загрузки.
- При отключённой сети расписание всё равно отображается (из кеша).
- В пикере настроек доступен актуальный список групп с ЯГТУ-API.

## Scope Boundaries

- UI экрана настроек (пикер групп/преподавателей) — **вне скоупа**, реализуется позже.
- Push-уведомления и фоновый polling — **вне скоупа**.
- Авторизация / OAuth — API публичный, не требуется.

## Key Decisions

- **Список групп кешируется в Drift** (не только SharedPreferences): нужен offline-доступ к пикеру.
- **Список преподавателей кешируется в Drift**: аналогично.
- **Расписание фильтруется по чётности на стороне клиента**: API возвращает все занятия семестра, клиент фильтрует по текущей/выбранной неделе.
- **Режим преподавателя включён в скоуп** (endpoint и кеш расписания преподавателя).

## Dependencies / Assumptions

- `dio_client.dart` нужно настроить (baseUrl, таймауты) перед реализацией `ApiClient`.
- Существующая Drift-схема (`schedule_dao`, `drift_database`) будет расширена новыми таблицами для групп и преподавателей.
