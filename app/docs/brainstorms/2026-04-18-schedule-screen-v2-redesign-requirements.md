# Schedule Screen v2 Redesign

## Problem Frame

Приложение должно соответствовать целевому макету (Image 1). Текущий экран расписания (Image 2) отличается по четырём направлениям: нет персонализированного заголовка, WeekStrip содержит явные кнопки-стрелки, отсутствует метка секции с количеством пар, карточки занятий держат время внутри (тогда как в макете — внешняя левая колонка), а тип занятия смещён влево вместо правого угла карточки.

## Requirements

**Заголовок экрана расписания (новый)**

- R1. Добавить персонализированное приветствие "Привет, [Имя]! 👋" крупным жирным текстом в верхней части экрана расписания, над `SyncStatusBar`.
- R2. Под приветствием — строка с текущей датой в формате "День недели, D месяца" (напр. "Среда, 19 марта"). Дата всегда означает сегодня, не следует за выбранным днём в WeekStrip.
- R3. Иконка уведомлений (колокольчик) прижата к правому краю строки с приветствием. Без обработчика нажатия в рамках этого скоупа.
- R4. Имя берётся из `Profile`: `displayName` для студента, первое слово `teacherName` для преподавателя. Если имя не задано — "Привет! 👋" без имени. Используется `profileNotifierProvider` (уже реализован).

**WeekStrip — навигация**

- R5. Убрать кнопки-стрелки (`IconButton` с `Icons.chevron_left` / `Icons.chevron_right`) из `WeekStrip`. Навигация между неделями остаётся только жестом свайпа (уже реализован `onHorizontalDragEnd`).

**Метка секции**

- R6. Перед списком занятий показывать строку-метку с количеством пар:
  - Для текущего дня: `СЕГОДНЯ • N ПАР`
  - Для другого дня: `ПН, 20 АПРЕЛЯ • N ПАР` (сокращение дня недели заглавными + дата)
  - N = количество занятий в выбранный день (длина списка `lessons`).

**Карточка занятия — временна́я колонка**

- R7. Время занятия переносится из тела карточки во внешнюю левую колонку на уровне `TimelineItem` (`TimelineItem` перестраивается: к текущему `ConstrainedBox(LessonCard)` добавляется фиксированная колонка ~52 dp слева — итоговая структура `Row([time column, Expanded(LessonCard)])`). Время начала — крупнее/темнее, время окончания — мельче/серее, выровнены по левому краю.
- R8. `LessonCard` больше не рендерит строку времени (удаляется только виджет `Text` с диапазоном начала–конца; `ref.watch(nowProvider)`, переменные `isActive` и `isPast` остаются). Карточка занимает всю оставшуюся ширину строки `TimelineItem`.
- R9. Внутри карточки, строка 1: название предмета (`Expanded`) слева + бейдж типа занятия (`_TypeBadge`) справа — в одной `Row`. Текущее размещение бейджа левее названия отменяется.

**Карточка занятия — статус прошедшего занятия**

- R10. Для прошедших занятий (`isPast`) добавить бейдж "Завершено" (серый или зелёный с галочкой) отдельной строкой под строкой названия предмета. Решение о сохранении/удалении `Opacity(0.48)` принимается при планировании.

## Success Criteria

- Экран расписания показывает приветствие с именем пользователя и датой.
- WeekStrip не имеет кнопок-стрелок; смена недели — только свайпом.
- Над списком занятий видна метка "СЕГОДНЯ • N ПАР" / "ДД, D МЕС • N ПАР".
- Время занятия отображается в левой колонке вне карточки; карточка не содержит строку времени.
- Тип занятия (ЛЕКЦИЯ, ЛАБ. РАБОТА и т.д.) отображается в правом верхнем углу карточки.
- Для прошедших занятий виден бейдж "Завершено".

## Scope Boundaries

- Функциональность уведомлений (колокольчик — только иконка без действия).
- Изменение цветовой схемы карточек и полосы слева.
- Изменение формата данных (`teacher`, `room`) — визуальное различие макетов объясняется тестовыми данными.
- Анимации появления/исчезновения карточек.
- Нижняя навигационная панель.
- Онбординг и профиль вне экрана расписания.

## Key Decisions

- **Внешняя временна́я колонка**: пользователь выбрал вернуться к колонке слева от карточки (отменяет R1 из `2026-04-18-lesson-card-redesign-requirements.md`).
- **Тип занятия справа**: перемещение бейджа в правый угол даёт карточке «газетный» вид — тип не перебивает взгляд при сканировании названия.
- **Без стрелок в WeekStrip**: свайп достаточен для навигации; стрелки занимают место и дублируют жест.
- **Граничные недели**: `onHorizontalDragEnd` в `WeekStrip` уже проверяет `canGoPrev`/`canGoNext` перед вызовом `_shiftWeek` — граничные случаи корректно обрабатываются и после удаления кнопок-стрелок.

## Dependencies / Assumptions

- `profileNotifierProvider` читает профиль из SharedPreferences — уже реализован. Имя пользователя доступно через `Profile.displayName`.
- `LessonProgressBar` уже отображает "Прошло X мин" / "Осталось X мин" — изменений не требует.
- `ActiveBadge` остаётся без изменений.
