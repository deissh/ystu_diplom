# Lesson Card Redesign

## Problem Frame

Карточки занятий выглядят компактно и информационно бедно. Пользователь хочет более просторный, визуально «свободный» дизайн с бо́льшим количеством информации. Аватарки преподавателей не несут смысловой нагрузки (берутся буквы фамилии и имени, а не фамилии и отчества). Верстка не проверена на широких экранах (планшеты).

## Requirements

**Компоновка карточки**

- R1. Время начала и окончания занятия отображается внутри карточки в формате `09:50 – 11:25`, а не в отдельной внешней колонке слева.
- R2. `TimelineItem` убирает колонку с временем; карточка занимает всю доступную ширину.
- R3. Внутренние отступы карточки увеличиваются, чтобы контент дышал (более просторный вид). Минимальная высота карточки растёт органически за счёт отступов.
- R4. Порядок элементов внутри карточки сверху вниз: время → [бейдж типа + название предмета] → [аватарка преподавателя + аудитория]. Для активного занятия ниже — `ActiveBadge` + `LessonProgressBar`.

**Аватарка преподавателя**

- R5. Инициалы в аватарке формируются из **фамилии** (первая буква) и **отчества** (первая буква), а не фамилии и имени. Оба распространённых формата поля `teacher` должны давать корректный результат:
  - «Иванов А.В.» → «ИВ»: первая буква слова 0 (фамилия) + первая буква слова 2 после разбивки строки по `[. ]` (отчество).
  - «Иванов Александр Владимирович» → «ИВ»: первая буква слова 0 + первая буква слова 2 (через пробел).
  - В обоих случаях алгоритм: взять первую букву первого токена (до пробела) + первую букву третьего токена при разбивке по `[ .]`. Если третий токен недоступен — достаточно одной буквы фамилии.
- R6. Размер аватарки увеличивается до 28–32 dp (вместо текущих 22 dp); точное значение определяется при реализации исходя из новых отступов.

**Адаптивность**

- R7. На узких экранах (< 360 dp) все элементы умещаются без горизонтального переполнения: текст предмета переносится на 2 строки максимум, имя преподавателя обрезается с `ellipsis`.
- R8. На широких экранах (≥ 600 dp) ширина контентной области ограничивается на уровне `TimelineItem` через `AppLayout.maxContent(context)` (680 dp при 600–899 dp, 720 dp при ≥ 900 dp) и горизонтальные отступы `AppLayout.hPad(context)`. Сам `LessonCard` не знает о ширине экрана.

## Success Criteria

- Карточка визуально стала выше и просторнее без потери читаемости.
- Время занятия легко читается прямо в карточке.
- Аватарка отображает «ИВ» для «Иванов А.В.», а не «ИА».
- На типичном узком экране (≤ 360 dp) и типичном широком (≥ 600 dp) нет горизонтального переполнения и некорректного переноса.

## Scope Boundaries

- Не меняем цветовую схему карточек и полосу слева.
- Не вводим новые поля в сущность `Lesson` (корпус, подгруппа и т.п.) — только переоформляем уже существующие данные.
- Не трогаем `WeekStrip`, `BreakRow`, `SyncStatusBar`.
- Анимации карточек (появление, свайп) — вне скоупа.

## Key Decisions

- **Время внутри карточки**: пользователь выбрал этот вариант, чтобы карточка была самодостаточной единицей без внешней временной колонки.
- **Инициалы из фамилии + отчества**: смысловой выбор — фамилия + отчество дают более уникальную пару, чем фамилия + имя в сокращённом формате.
- **Без новых полей данных**: карточка переоформляется визуально, данные не расширяются.

## Dependencies / Assumptions

- Формат поля `teacher` в `Lesson` остаётся «Фамилия И.О.» (как в текущих моковых данных). Алгоритм инициалов должен поддерживать оба варианта: сокращённый («Иванов А.В.») и полный («Иванов Александр Владимирович»).
