# Вкладка «Календарь»

## Problem Frame

Пользователь хочет видеть расписание в контексте месяца — быстро найти нужный день, оценить нагрузку по неделям, и сразу посмотреть пары. Вкладка «Расписание» даёт только недельный вид; месячного обзора нет.

## Requirements

- **R1.** В нижней навигации появляется новая вкладка «Календарь» на позиции 2 (между Расписанием и Профилем), заменяя вкладку Настройки. Итоговый порядок: `Расписание | Календарь | Профиль`.
- **R2.** Экран «Календарь» показывает месячный календарь с возможностью навигации по месяцам (стрелки назад / вперёд). По умолчанию открывается текущий месяц.
- **R3.** В каждой ячейке дня отображаются цветные точки — по одной точке на каждую пару. Цвет определяется типом занятия: лекция — пурпурный, практика — голубой, лаборатория — зелёный, прочее — серый (соответствует `LessonType` и цветам из `AppColors`).
- **R4.** Под календарём расположен виджет-легенда, поясняющий соответствие цветов типам занятий (все четыре типа с подписями).
- **R5.** Тап по дню в календаре выбирает его; timeline ниже обновляется и показывает пары на этот день. По умолчанию выбран сегодняшний день.
- **R6.** Выбранный день в Календаре хранится в отдельном провайдере (независимо от `selectedDayProvider` вкладки Расписание). Переключение между вкладками не сбрасывает выбранный день.
- **R7.** Компоновка экрана: календарь и легенда зафиксированы вверху, timeline прокручивается независимо снизу.
- **R8.** Timeline переиспользует существующие виджеты (`TimelineItem`, `BreakRow`) и обрабатывает пустые состояния: выходной день, нет пар, ошибка загрузки — аналогично `ScheduleScreen`.

## Success Criteria

- Пользователь может открыть любой месяц, выбрать день и сразу увидеть расписание на него без перехода на другую вкладку.
- Цветные точки на ячейках дней позволяют за секунду оценить нагрузку и типы занятий.
- Легенда однозначно объясняет, что означает каждый цвет.
- Состояние Календаря сохраняется при переключении между вкладками.

## Scope Boundaries

- Настройки убираются из навигации (вкладка была заглушкой без реализованного роута).
- Точки показываются по одной на пару, не агрегируются по типу.
- Нет возможности добавлять / редактировать события прямо из Календаря.
- Нет перехода с Календаря на вкладку Расписание при выборе дня (вкладки независимы).

## Key Decisions

- **Позиция вкладки**: 3-я позиция (замена Настроек) — Настройки всё равно заглушка, 3 вкладки чище чем 4.
- **Цвет точек по типу занятия, не по предмету**: тип занятия стабилен и сразу виден в легенде; цвет предмета зависит от названия и менее предсказуем.
- **Одна точка на пару**: даёт полную картину нагрузки за день; агрегация по типу скрывала бы количество пар.
- **Независимое состояние дня**: Calendar и Schedule — разные сценарии использования, общий провайдер создал бы нежелательные побочные эффекты.
- **Фиксированный календарь + прокручиваемый timeline**: пользователь всегда видит месяц и может скроллить список пар, не теряя контекст.
